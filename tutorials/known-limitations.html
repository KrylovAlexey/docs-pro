Although we are working hard on this library, it still has some known shortcomings. Consider this list to make sure there are no potential blockings.

  • Formula support is based on a third-party library

    We use formula.js to implement functions. That library is no longer under active development, but we believe it's reliable enough to be utilized even in its current state.

  • It is not mobile-friendly (yet)

    Only the basic functionality works with mobile devices, which means you should be able to display, edit and scroll the grid. We can't guarantee that all the plugins will work as we don't test Handsontable against mobile browsers.

  • It is not available via npm/NuGet

    There is no officially supported package in npm or NuGet directories. To facilitate the installation we usually use Bower.

  • It uses several dependencies

    Our team follows the "proudly found elsewhere" principle which encourages us to make use of the great work done by other developers. You will probably find the following open source libraries being utilized by Handsontable:

    • numeral.js (handles numeric data)
      Author: Adam Draper
      License: Open source (MIT)
      http://adamwdraper.github.com/Numeral-js
    • ZeroClipboard (allows to copy and paste data using the context menu)
      Author: Jon Rohan, James M. Greene
      License: Open source (MIT)
      http://zeroclipboard.org
    • Pikaday (displays a date picker)
      Author: David Bushell
      License: Open source (BSD & MIT)
      https://github.com/dbushell/Pikaday
    • moment.js (parses, validates and displays dates)
      Author: Tim Wood, Tskren Chernev, Moment.js contributors
      License: Open source (MIT)
      http://momentjs.com
    • formula.js (provides syntax and logic for functions)
      Author: Sutoiku, Inc
      License: Open source (MIT)
      http://stoic.com

  • It is not fully covered by unit tests

    In the current version only a part of the code is covered by unit tests. We aim to increase that number to at least 90%.

  • Copy-paste is dependent on Flash

    We use ZeroClipboard to allow users copy and paste using the context menu. That library uses Flash to make it possible. We are seriously considering switching to Clipboard API as soon as it's fully supported by all popular browsers.
